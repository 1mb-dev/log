Building is addition. Solo building is subtraction.

A user asked for dark mode. Reasonable. Useful. Probably an afternoon's work. Not built.

A platform offered to feature the project if integrations were added for AI agents. Reasonable. Probably a week. Not built.

A friend pitched a Slack bot version. Sharp idea. Two weekends, max. Not built.

Each one was a real opportunity. Each would have shipped fine. Each would have made the next one slower.

Everything added has a second cost. The docs, the bug reports six months later, the support thread from a feature added in a single afternoon and forgotten by Friday. Adding is fast. Owning is forever.

The first feature is enthusiasm.
The fifth is rent.
The tenth is a project that maintains its features more than it serves its users.

Saying yes is generous. Open. Roadmap-friendly.
Saying no looks like neglect. Until something ships.

The thing nobody publishes is the rejected list. The features evaluated and declined. The integrations passed on. The collaborations let go. That list is most of the actual work.

Most builders can produce. Few can subtract.
The projects that feel coherent are subtracting in private.

Every no is a quiet decision the project still knows what it is.

The shipped thing is the visible part.
The rejected list is the spine.
